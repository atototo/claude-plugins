# 콘텐츠 분석 가이드

## 토픽 vs 엔티티 판단

- **토픽**: 기술/트렌드/개념 (예: ai-agent, fine-tuning, prompt-engineering, rag)
- **엔티티**: 회사/인물/제품 (예: anthropic, ethan-mollick, claude-code)
- 경계가 모호하면 토픽 우선. 예: "MCP"는 토픽, "Claude"는 엔티티

## 엔티티 노트 생성 기준

**엔티티 노트는 팔로우 여부와 완전히 무관하다.** Phase 4 팔로우 기준(팔로워 5000+)과 혼동 금지.

**엔티티 노트 생성 조건 (하나라도 해당하면 생성):**
- 이번 리서치에서 **처음 등장**한 인물/회사/제품이고 AI 분야에서 주목할 만한 경우
- 여러 포스트에서 언급된 인물/회사/제품
- 딥다이브 대상 포스트(❤️50+)의 작성자
- 인플루언서 타임라인(Phase 2)에서 직접 언급된 인물/프로젝트

**엔티티 노트 생성 예시:**
- `@beamt_io` — Preflight API 개발, AI 에이전트 인프라 → ✅ 엔티티 노트 생성
- `@KeycardLabs` — identity-based authz 선도 → ✅ 엔티티 노트 생성
- Anthropic — Claude 관련 포스트 다수 → ✅ 엔티티 노트 생성
- 일회성 언급 + AI 무관 인물 → ❌ 생성 불필요

**엔티티 노트 ≠ 팔로우.** 팔로우 기준 미달이어도 엔티티 노트는 만든다.

## 독립 토픽 생성 기준

기존 토픽에 하위 항목으로 넣을지, 독립 토픽으로 만들지 판단하는 기준:

**독립 토픽 생성 (적극적으로):**
- engagement ❤️100+ 이고 기존 토픽과 직접적 상위-하위 관계가 아닌 경우
- 명확한 도구/프로젝트/제품명이 있는 경우 (GitHub 레포, SaaS 제품 등)
- 기존 토픽의 하위 항목이 3개 이상 쌓인 경우 → 독립 토픽으로 분리
- 2개 이상의 독립적 포스트에서 언급된 경우

**기존 토픽에 포함:**
- engagement ❤️50 미만이고 기존 토픽의 사례/응용인 경우
- 일회성 언급으로 지속성이 불확실한 경우
- 기존 토픽의 구체적 사용 사례인 경우

**판단 예시:**
- "Skales" (❤️189, 명확한 제품명) → ✅ 독립 토픽 `skales.md`
- "PUA coding" (❤️130, 8.9K stars 프로젝트) → ✅ 독립 토픽 `pua-coding.md`
- "ClawRouter" (❤️-, LLM 라우터) → ✅ 독립 토픽 `clawrouter.md`
- "Claude 프롬프트 8레이어" (❤️144, 일반적 팁) → ❌ `claude-code.md`에 포함
- "AI 코딩 병목은 인간" (❤️-, 의견) → ❌ `ai-coding-tools.md`에 포함

**원칙: 토픽은 적극적으로 만들되, 내용이 얇으면 나중에 합칠 수 있다. 반대로 묻혀버린 토픽은 발견하기 어렵다.**

## 수집 로그 작성 (유실 방지)

**모든 수집 포스트를 runs 파일의 수집 로그 테이블에 기록한다.** 토픽/엔티티 노트에 반영되지 않은 포스트도 빠짐없이.

```markdown
| # | 출처 | 작성자 | 핵심 내용 (1줄) | ❤️ | 노트 반영 |
|---|------|--------|----------------|-----|----------|
| 1 | Phase 0 | @user | 내용 요약 | 1.2K | ✅ [[토픽명]] |
| 2 | Phase 1 | @user2 | 내용 요약 | 89 | ❌ (중요도 낮음) |
| 3 | Phase 1 | @user3 | 내용 요약 | 230 | ✅ [[토픽명]] 🆕 |
```

- `노트 반영` 열: ✅ 반영된 노트 wikilink / ❌ 미반영 사유
- 🆕 는 이번 실행에서 새로 생성된 토픽
- 이 로그로 "47개 수집 → 25개만 노트화"의 갭을 추적 가능

**핵심: 수집 로그의 행 수 = 실제 수집된 포스트 수. 누락 금지.**

## 토픽 노트 품질 기준 (학습 가능한 수준)

토픽 노트는 **이것만 읽고도 해당 기술/트렌드를 이해할 수 있는 수준**이어야 한다. 단순 뉴스 요약이 아니라 학습 자료.

**필수 요소:**
1. **구체적 설명**: "오픈 CLI 도구"가 아니라 "pip install chub로 설치, chub add <library>로 문서 등록, 에이전트가 chub query로 최신 API 문서 검색" 수준
2. **외부 리소스 링크**: GitHub 레포, 공식 사이트, 코스 URL, 논문 링크 등. 포스트에 링크가 있으면 반드시 수집. 없으면 Phase 3 리플라이에서 탐색
3. **작동 원리/구조**: 어떻게 동작하는지, 어떤 기술 스택인지, 어떤 문제를 해결하는지
4. **실용 정보**: 설치 방법, 가격, 제한사항, 호환성

**`## 리소스` 섹션 필수:**
```markdown
## 리소스
- [GitHub](https://github.com/...)
- [공식 사이트](https://...)
- [코스](https://deeplearning.ai/...)
- [논문](https://arxiv.org/...)
```
- 포스트 본문, 리플라이, 프로필에서 발견되는 모든 관련 링크를 수집
- 링크가 없으면 Phase 3-4에서 WebFetch로 탐색
- **추측으로 링크를 만들지 않는다** — 없으면 "링크 미발견"으로 기록

**나쁜 예 (현재):**
```markdown
**Context Hub 발표** ❤️4.9K
Andrew Ng가 Context Hub(chub) 공개. 코딩 에이전트에 최신 API 문서를 제공하는 오픈 CLI 도구.
```

**좋은 예 (목표):**
```markdown
**Context Hub 발표 — AI agent 간 지식 공유 인프라** ❤️4.9K
Andrew Ng가 Context Hub(chub) 공개. AI 코딩 에이전트가 outdated 문서로 코드를 생성하는 문제를 해결.

핵심 구조:
- 오픈 CLI 도구 — `npm install -g chub`로 설치
- 라이브러리 관리자가 `chub add`로 최신 API 문서를 등록
- 에이전트가 `chub query <library>`로 실시간 문서 검색
- agent-to-agent 지식 공유: 한 에이전트가 발견한 패턴을 다른 에이전트가 활용

> "Should there be a Stack Overflow for AI coding agents to share learnings with each other?" — @AndrewYNg
([원문](https://x.com/AndrewYNg/status/...))
```

## 원문 인용구 포함 (RAG 적중률)

토픽 노트의 타임라인 항목에 **핵심 문장 1-2개를 `> ` 인용 블록으로 포함**한다.

```markdown
**Skales: 300MB RAM 데스크톱 AI 에이전트** ❤️189
@hasantoxr 소개. 터미널 없이 30초 설치.
> "You can now run a full AI agent on your desktop without touching a terminal once" — @hasantoxr
([원문](https://x.com/hasantoxr/status/...))
```

**인용구 규칙:**
- 원문 그대로 옮기되 핵심 부분만 발췌 (1-2문장)
- 벡터 검색 시 원문 표현과 매칭 → 적중률 향상
- 모든 포스트에 인용구를 넣을 필요는 없음. 핵심 포스트(❤️100+)에 우선 적용
- 이미지 포스트에서 추출한 텍스트도 인용구로 포함 가능
- 작성자 attribution 필수: `— @handle` 또는 `— 이름`

## 키워드 자동 발견

수집된 포스트 전체에서 자주 등장하지만 현재 키워드 목록에 없는 용어를 찾는다:

- 2개 이상 포스트에서 반복 등장하는 새 용어
- 예: "Cowork" 반복 → `discovered`에 추가
- 예: "AgentPay" 새 트렌드 → `discovered`에 추가
- 설정 파일의 `keywords.discovered` 배열에 자동 추가
- 다음 실행 시 Phase 1에서 이 키워드로도 검색

## 중복 제거

- URL 기준으로 중복 포스트 제거
- 같은 포스트가 Phase 0과 Phase 1에서 모두 발견되면 먼저 수집된 쪽을 유지
- 리트윗은 원문 포스트와 URL이 같으므로 자동 중복 제거

## 중요도 정렬

engagement(❤️) 기준으로 정렬. 딥다이브 우선순위에도 사용:
1. ❤️1,000+ → 최우선 딥다이브
2. ❤️100-999 → 딥다이브 대상
3. ❤️50-99 → 선택적 딥다이브 (콘텐츠 가치에 따라)
4. ❤️10-49 → 수집 로그에 기록, 노트 반영은 선택
5. ❤️10 미만 → 수집 대상 아님 (min_engagement 설정)
