# Phase 상세 절차

## Phase 0: 메인 피드 스캔 (유기적 발견)

키워드 검색보다 먼저 X 메인 피드를 확인한다. 팔로우한 사람들의 최신 포스트에서 **아직 키워드화되지 않은 새로운 트렌드**를 발견하는 것이 목적. 키워드 검색은 "이미 알고 있는 것"만, 메인 피드는 "아직 모르는 것"을 찾는다.

**Following 탭과 For you 탭 모두 스캔한다.**
- **Following 탭**: 팔로우한 사람들의 최신 포스트 (시간순, 놓침 없음)
- **For you 탭**: X 알고리즘이 추천하는 인기/트렌드 포스트 (더 넓은 발견)

**절차:**

### 0-1. Following 탭 스캔
1. `navigate`로 `x.com/home` 이동
2. Following 탭 클릭 (이미 Following이면 스킵)
3. 4초 대기
4. `javascript_tool`로 포스트 추출 (references/scraping.md 참조)
5. 스크롤 → 3초 대기 → 재추출 (최대 3회, 30-40개 목표)
6. AI 관련 포스트 필터링

### 0-2. For you 탭 스캔
1. For you 탭 클릭
2. 4초 대기
3. `javascript_tool`로 포스트 추출
4. 스크롤 → 3초 대기 → 재추출 (최대 3회, 20-30개 목표)
5. AI 관련 포스트 필터링
6. Following 탭에서 이미 수집한 포스트는 URL 기준 중복 제거

**가치:**
- **Following**: 팔로우한 인플루언서의 리트윗/인용에서 새로운 인물+토픽 발견
- **For you**: 팔로우하지 않은 계정의 인기 포스트에서 더 넓은 트렌드 포착, 새로운 인플루언서 후보 발견
- 기존 키워드에 없는 신규 용어 자연스럽게 포착
- 발견된 새 키워드는 Phase 1 검색어에 즉시 추가

**피드 품질은 팔로우에 의해 결정된다.** 이것이 Phase 4 자동 팔로우가 중요한 이유.

---

## Phase 1: 키워드 검색 (넓게 훑기)

각 키워드마다 별도 검색 URL로 이동하여 수집.

**절차 (키워드당):**
1. `navigate`로 `x.com/search?q={keyword}%20min_faves:{min_engagement}&src=typed_query&f=live` 이동
2. 4초 대기
3. `javascript_tool`로 포스트 추출 (references/scraping.md 참조)
4. 스크롤 → 3초 대기 → 재추출 → 새 포스트 없으면 중단 (최대 3회)
5. 다음 키워드 전 3초 딜레이

**포스트 전문(500자까지)**과 engagement 수치를 수집한다.

---

## Phase 2: 인플루언서 타임라인 탐색 (깊게 보기)

키워드 검색만으론 놓치는 중요 콘텐츠가 많다. AI 분야 인플루언서의 최근 포스트를 직접 확인.

**절차 (인플루언서당):**
1. `navigate`로 `x.com/{handle}` 이동
2. 4초 대기
3. `javascript_tool`로 최근 포스트 10개 추출
4. AI 관련 포스트만 필터링
5. 다음 인플루언서 전 3초 딜레이

**인플루언서 자동 발견:**
Phase 1에서 engagement 높은(likes > 50) 포스트의 작성자를 확인:
- bio/프로필에서 AI 관련 인물인지 판단
- AI 관련이면 `influencers` 목록에 추가 → Phase 2에서 탐색
- 새로 발견된 인플루언서는 설정 파일에 자동 추가 (다음 실행에도 추적)

---

## Phase 3: 딥다이브 (고반응 포스트 깊이 파기)

Phase 0-2에서 engagement가 `deep_dive_threshold`(기본 50) 이상인 포스트를 선별하여 깊이 탐색. **이 단계가 벡터화 시 진짜 가치를 만드는 핵심.**

**딥다이브 대상:**
- likes >= `deep_dive_threshold`
- 스레드(1/n, 2/n)의 첫 포스트
- 외부 링크(블로그, 논문, GitHub)가 포함된 포스트

### 3-1. 스레드 전체 읽기

- 포스트 URL로 `navigate` → 4초 대기
- `javascript_tool`로 원 작성자의 연속 답글 전체 추출
- 스레드 전문을 하나의 텍스트로 결합

### 3-2. 미디어 포스트 처리

텍스트가 비어있거나 짧은 경우 (이미지/영상 기반 포스트):

- `javascript_tool`로 미디어 타입 감지: `article.querySelector('img[src*="media"]')`, `article.querySelector('video')`
- **이미지/인포그래픽**: `computer(action: "screenshot")` 또는 `computer(action: "zoom", region: [이미지영역])` → Claude 멀티모달로 읽어 핵심 내용 추출. 데이터, 비교표, 다이어그램을 텍스트로 변환
- **캐러셀 (다중 이미지)**: 각 이미지마다 스크린샷. `computer(action: "left_click")`으로 다음 이미지 버튼 클릭
- **영상**: 썸네일 스크린샷 + 리플라이에서 요약/타임스탬프 코멘트 수집. 영상 시청은 불가
- 미디어 추출 내용은 `[이미지에서 추출]` 또는 `[영상 썸네일/리플라이 기반]` 태그 부착

### 3-3. 핵심 리플라이 수집 & 링크 발굴

- 같은 포스트 페이지에서 engagement 높은 리플라이 상위 3-5개 추출
- 전문가의 반론, 보충 정보, 관련 링크가 특히 가치 있음
- **리플라이에서 GitHub/공식 링크 탐색**: 원문에 프로젝트/도구/논문이 언급되었지만 링크가 없는 경우, 리플라이에서 GitHub URL, 공식 사이트, 논문 링크 탐색
- 오픈소스 프로젝트(GitHub stars 언급 등)는 링크가 노트의 실용성에 핵심 → 반드시 리플라이까지 확인
- 찾은 링크는 `[GitHub](url)` 또는 `[공식](url)` 형태로 기재
- **링크를 찾지 못한 경우 추측으로 링크를 만들지 않는다** (벡터 오염 방지)

### 3-4. 외부 링크 콘텐츠 수집 (필수)

**이 단계가 토픽 노트의 학습 가능성을 결정한다.** 포스트 텍스트만으로는 "~가 ~를 출시했다" 수준의 뉴스 요약밖에 안 된다. 외부 링크에서 구체적 정보를 가져와야 실제로 배울 수 있는 노트가 된다.

**절차:**
1. 포스트 텍스트에서 URL 추출 (`javascript_tool`로 `a[href]` 탐색)
2. 리플라이에서 추가 URL 탐색 (3-3에서 수집)
3. 발견된 URL을 `WebFetch`로 수집

**수집 대상 & 추출 정보:**
- **GitHub**: 프로젝트 설명, stars, 주요 기능, 설치 방법(`npm install`, `pip install` 등), 기술 스택 → 토픽 노트의 `## 리소스`에 기재
- **논문(arXiv 등)**: 제목, 저자, Abstract 요약, 핵심 발견 → 토픽 노트에 구체적 기술 내용 반영
- **블로그/기사**: 핵심 주장, 구체적 수치/벤치마크, 코드 예시 → 토픽 노트의 타임라인 항목에 반영
- **코스/튜토리얼**: 제공자, URL, 커리큘럼 구성, 수준 → `## 리소스`에 기재
- **제품 사이트**: 가격, 기능 목록, 호환성 → 토픽 노트의 설명에 반영

**수집한 링크는 토픽 노트의 `## 리소스` 섹션에 반드시 기재한다.**
링크를 찾지 못한 경우 추측으로 만들지 않는다.

### 3-5. 작성자 프로필 확인

딥다이브 대상 포스트 작성자가 아직 엔티티 노트에 없으면:
- `x.com/{handle}` 방문 → bio, 소속, 팔로워 수 확인
- AI 관련 인물이면 엔티티 노트 생성

---

## Phase 4: 신뢰 기반 자동 팔로우 & 피드 확장

Phase 0-3에서 **팔로우할 가치가 있는 AI 인물**을 식별, 조건 충족 시 자동 팔로우. 다음 실행의 Phase 0에서 그 사람의 포스트가 나타나 **리서치 범위가 복리로 확장**.

**자동 팔로우 기준** (모두 충족):
1. AI 관련 포스트 3개 이상
2. AI 포스트 평균 engagement 100 이상
3. 팔로워 5,000명 이상
4. 아직 팔로우하지 않은 계정

기준값은 설정의 `trusted_follows.follow_threshold`에서 조정.

**절차:**
1. Phase 0-3에서 여러 번 등장 / high-engagement 작성자를 후보 선별
2. 프로필 방문 → `javascript_tool`로 팔로워 수, bio 확인
3. 충족 시: 팔로우 버튼 클릭 (`[data-testid="follow"]`), 설정에 기록, daily 노트에 기록
4. 미충족 시: `influencers` 목록에만 추가 (팔로우 없이 탐색만)

**`auto_follow: false`이면** 팔로우 추천만 기록하고 실제 팔로우는 하지 않는다.

**선순환 구조:**
```
메인 피드 스캔(Phase 0) → 새 인물 발견 → 기준 충족 시 팔로우(Phase 4)
    ↑                                                          ↓
    └──── 다음 실행 시 피드가 더 풍부해짐 ←───────────────────────┘
```
